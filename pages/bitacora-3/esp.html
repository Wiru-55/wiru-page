ESPECIFIDAD

IDEAS CLAVE

  • Jerarquía:
    Inline > ID > Class-att-psclass > Elementos

  • Regla:
    Si se escribe 2 veces en la mismo CSS, solo aceptara el cambio mas reciente de entre ellos 2.

  • Selector Universal:
    Si alguna elemento no tiene una referencia en CCS y esta aplicado el *, entonces estos elementos seran cambios por el selector universal, todos los elementos, ids, classes, etc, tienen todos una especifidad de 0, es decir, todos pueden recibir cambios si no tienen una referencia ya establecida anteriormente en el CSS.

  • Limportant:
    Permite realizar cambios y anularlos, esto puede ser un poco util para hacer cambios repentidos o de emergencia.

NOTAS DE LA CLASE

Es el medio por el cual los navegadores deciden que los valores de propiedad de CSS son los mas relevantes para un elemento, y por lo tanbto se aplicaran.

- Como se calcula

  • Solo se aplica cuando el mismo elemento es el objetivo de varias declaraciones. Segun las reglas de CSS, los elementos dirigidos directamente siempre tendran la prioridad sobre las reglas que un elemento hereda de su antepasado.
  • Cuando varias declaraciones tienen la misma especificidad, la ultima declaracion encontrada en CSS se aplica el elemento.
- Jerarquía
Cada selector tiene un lugar en la jerarquia de especifidad. Hay cuatro categorias que definen el nivel de especifidad de un selector:
  1. Estilos Inline:
    Se adjunta directamente al elemento que se va a diseñar
    < h2 style=" color:white ">

  2. ID:
    identificador unico para elementos de la pagina
    #navbar

  3. Class, atributos y PseudoClases:
    hover,focus, etc.

  4. Elementos y pseudoelementos
    Esto incluye nombres de elementos y pseudo elementos:
    h1, nav, section, etc.
- Regla
si la misma regla se escribe dos veces en la hoja de estilo externa, entonces la regla inferior en la hoja de estilo esta mas cerca del elemento que se va a diseñar y por lo tanto, se aplicara.

- Selector Universal
El selector universal y los valores heredados tienen una especificad de 0 - *, el cuerpo * y similares tiene una especificad cero. Los valores heredados también tienen una especificad de 0.

- Limportant
  • Cuando se usa una regla limportant es una declaración de estilo, esta declaración anula cualquier otra declaración.

  • Aunque técnicamente limportant no tiene nada que ver con la especifidad, interactua directamente con ella

  • Sin embargo, usar limportant es una mala practica y debe evitarse porque dificulta la depuración al romper la cascada natural en sus hojas de estilo

  • Cuando se aplican dos declaraciones en conflicto con la regla limportant al mismo elemento, se aplicará la declaración con una mayor especifidad.

RESUMEN

Gracias a la especifidad en CSS nos permite decidir que estilos se aplicaran a un elemento, ya que por este, realiza una jerarquia de cambios, ya sea estilos en linea, ID, classes, atributo, pseudoclasses y elementos.

Con las reglas y el limportant nos permite anular otras, pero esto no se aconseja utilizarse por lo problematico que puede resultar mas adelante.

En si la especifidad nos permite evitar conflictos y mantener el orden en los estilos de una pagina web.
